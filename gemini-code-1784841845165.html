IPv6 Address Compression

Networking 101: Learn to shorten 128-bit IPv6 addresses accurately.

Learning Objectives

  • Remember: Recall the two official IETF rules for compressing IPv6 addresses (RFC 5952).
  • Understand: Explain why double colons (::) can only be used once in an address.
  • Apply: Correctly compress full 8-hextet IPv6 addresses into their canonical shortest form.

Understanding IPv6 Addresses

An IPv6 address is 128 bits long, written as 8 groups of 4 hexadecimal digits separated by colons. Each group is called a hextet (or block).

Uncompressed IPv6 Address Example:

2001:0db8:0000:0000:0000:0000:0000:0001

Writing 32 hex digits gets long! We use two rules defined in RFC 5952 to shorten them without losing any information.

Rule 1: Omit Leading Zeros

In any 4-digit hextet, you can remove all zeros at the beginning of that block.

  • 0db8 becomes db8
  • 0005 becomes 5
  • 0000 becomes 0
Warning (Common Pitfall): Do not drop trailing zeros! 0500 becomes 500, not 5. Dropping trailing zeros changes the value.

Rule 2: Double Colon (::) Compression

You can replace a continuous sequence of two or more zero-hextets (e.g., :0000:0000:) with a single double colon (::).

Critical Rule: You can only use the double colon (::) ONCE per address! If you used it twice, a computer wouldn't know how many zero blocks belong to each side.

Fully Worked Example

Let's compress this full IPv6 address step-by-step:

2001:0db8:0000:0000:0008:0800:200c:417a
Step 1: Apply Rule 1 — Drop Leading Zeros

Look at each of the 8 hextets and drop all zeros that appear before other digits:

  • 2001 → 2001 (No leading zeros)
  • 0db8 → db8 (Drop 1 leading zero)
  • 0000 → 0 (Leave one zero)
  • 0000 → 0 (Leave one zero)
  • 0008 → 8 (Drop 3 leading zeros)
  • 0800 → 800 (Drop 1 leading zero; keep trailing zeros!)
  • 200c → 200c (No leading zeros)
  • 417a → 417a (No leading zeros)

Result after Step 1:

2001:db8:0:0:8:800:200c:417a
Step 2: Apply Rule 2 — Compress Consecutive Zero Hextets

Find the continuous sequence of zero blocks. Here we have two contiguous zero blocks: :0:0:.

Replace that group with a single ::.

Final Compressed Result:

2001:db8::8:800:200c:417a
RFC 5952 Best Practice: If there are two equal groups of zeros (e.g., two zero-blocks in one place and two zero-blocks in another), always compress the first group on the left.
