1. Anatomy of an IPv6 Address Remember / Understand

An IPv6 address consists of 128 bits written in hexadecimal format. It is divided into 8 groups of 4 hexadecimal digits (often called "hextets" or "blocks"), separated by colons.

Here is an example of an uncompressed (full format) IPv6 address:

2001 : 0db8 : 0000 : 0000 : 0008 : 0800 : 200c : 417a

Notice how long this address is! Writing full 39-character addresses increases the risk of typing mistakes. To make IPv6 easier to read and write, the Internet Engineering Task Force (IETF) defined two compression rules (RFC 5952).

2. The Compression Rules Understand

Rule 1: Omit Leading Zeros

In any hextet (4-digit group), you can drop all leading zeros (zeros at the start of a block).

  • 0db8 becomes db8
  • 0000 becomes 0
  • 0008 becomes 8
  • 0800 becomes 800 (Note: Trailing zeros stay!)
⚠️ Critical Pitfall: Never omit trailing zeros! 0800 becomes 800, NOT 8. Omitting trailing zeros completely changes the numerical value of the address.

Rule 2: Double Colon (::) Zero Compression

You can replace a single continuous sequence of two or more blocks of all zeros (:0:0:) with a double colon (::).

⚠️ Golden Rule: You may use the double colon (::) ONLY ONCE per IPv6 address! If used twice, it becomes impossible to determine how many zero blocks belong to each ::, rendering the address invalid.

3. Fully Worked Step-by-Step Example Apply

Let's compress the uncompressed address below step-by-step:

2001:0db8:0000:0000:0000:ff00:0042:8329
Step 1: Apply Rule 1 (Omit Leading Zeros)

Go through each hextet from left to right and remove leading zeros:

  • 2001 → 2001 (No leading zero)
  • 0db8 → db8
  • 0000 → 0
  • 0000 → 0
  • 0000 → 0
  • ff00 → ff00 (Keep trailing zeros!)
  • 0042 → 42
  • 8329 → 8329

Result after Rule 1:

2001:db8:0:0:0:ff00:42:8329
Step 2: Apply Rule 2 (Compress Contiguous Zero Blocks)

Identify the continuous run of all-zero blocks: :0:0:0:

Replace this sequence with a single ::.

Final Compressed Result:

2001:db8::ff00:42:8329
